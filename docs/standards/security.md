# Security & Privacy Standards

ローカルファースト（[ADR 0002](../adr/0002-local-first-with-sync-seam.md)）・匿名（[ADR 0003](../adr/0003-anonymous-auth-upgrade.md)）構成における、セキュリティとプライバシーの最低基準と改善提案。

## 脅威モデル（MVP）

- サーバーを持たないため、サーバー侵害・大規模漏洩の面は小さい。
- 主要リスクは **(a) 任意 URL を端末から fetch する OGP 取得**、**(b) 端末ローカルデータの保護**、**(c) 外部リンク遷移**。

## (a) OGP 取得の安全策（必須）

任意のユーザー入力 URL を fetch するため、SSRF / リソース枯渇を防ぐ（[save-flow](../specs/save-flow.md)）。

- **スキーム allowlist**: `http` / `https` のみ。`file:`/`data:`/`ftp:` 等は拒否。
- **プライベート/ループバック IP 拒否**: 解決先が `127.0.0.0/8` `10/8` `172.16/12` `192.168/16` `169.254/16` `::1` 等なら中止。
- **リダイレクト制限**: 最大 5 回。各ホップでも上記チェックを再適用。
- **サイズ上限**: レスポンス本文 2MB 程度で打ち切り。
- **タイムアウト**: 5 秒程度。
- **Content-Type 限定**: `text/html` のみ解析。それ以外は URL のみ保存。
- 取得した HTML はサニタイズして**メタタグの値だけ**抽出。HTML をそのままレンダリングしない（XSS 回避）。
- User-Agent は一般的なものを送り、Cookie や認証情報は付与しない。

## (b) ローカルデータ保護

- `owner_id` 等の識別子は `expo-secure-store`（Keychain / Keystore）に保管。
- 店舗・訪問データは個人の嗜好・行動履歴であり機微。**端末外に送らない**（MVP は完全ローカル）。
- **バックアップ/エクスポート**: 端末紛失でデータ消失するため、設定画面から手動 JSON エクスポート/インポートを提供する（[ADR 0002](../adr/0002-local-first-with-sync-seam.md)）。エクスポートは OS の共有シート経由でユーザー自身が保存先を選ぶ。
- 将来のリモート同期導入時に、何が送信されるかを明示し同意を取る。

## (c) 外部遷移

- 確定店舗の `source_url` を開く際、http/https のみ許可し、想定外スキームは開かない。
- ディープリンク（共有/通知）で受け取るパラメータは検証してから画面遷移に使う。

## プライバシー指針

- 計測/解析を入れる場合も、店舗の具体内容や URL を外部解析サービスへ送らない（イベント名・回数等の匿名集計に限定）。
- 通知許可・位置情報（将来）は、必要になった時点で目的を示して要求（[resurfacing](../specs/resurfacing.md)）。

## 改善提案（任意・優先度順）

1. エクスポート JSON の暗号化オプション（パスフレーズ）。
2. アプリ起動時の生体認証ロック（任意設定）。機微な行動履歴の保護。
3. OGP 取得を将来サーバー（Edge Function）に寄せると、端末 IP を晒さず一元的に SSRF 対策できる（リモート導入時）。
