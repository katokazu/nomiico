# ADR 0004: メタデータ取得は OGP + 連携元保存 + 手動補完

## Status

Accepted (2026-06-29)

## Context

共有された URL から店名・ジャンル・エリアなどを得たい。ただし無料運用が前提のため、Google Places API のような課金 API は当面使わない。スクレイピングは ToS リスクと壊れやすさが大きい。「3 秒保存」を壊さず、必要な人だけ後から詳細を埋められるようにしたい。

## Decision

段階的・ベストエフォートで取得し、**失敗しても URL は必ず保存する**。

1. **連携元 URL を常に保存する**（消えない一次情報）。`source_url` と `source_type` を必ず保持。
2. **OGP / メタタグ抽出**: 保存時に端末から対象 URL を `fetch` し、HTML の `og:title` / `og:site_name` / `og:image` / `og:description` を解析して店名・サムネイル・説明を補完する。RN ネイティブ fetch のため CORS 制約はない。
3. **oEmbed / 既知パターン（任意・軽量）**: 取得しやすいソースのみ簡易対応。Instagram は OGP がログイン無しで取りにくいため、**投稿 URL とサムネイル（取れれば）をそのまま保存**し、店名は手動に委ねる。
4. **手動補完**: 取得できなかった項目は後から手で埋められる。ただし入力は最小限・任意とし、保存をブロックしない。

Google Places / 地図 API は使わない。座標・営業時間・正確なジャンルは MVP では必須にしない。

## Consequences

- 課金ゼロでメタデータをある程度自動補完できる。
- 取得品質はソース依存でばらつく。特に Instagram は店名自動取得が弱く、手動前提になる。これは仕様として許容する（保存自体は速い）。
- 端末から任意 URL を fetch するため、**SSRF / 不正 URL / 巨大レスポンス対策**が要る（[security](../standards/security.md)）: スキーム allowlist（http/https のみ）、リダイレクト回数制限、サイズ・タイムアウト上限、HTML のみ解析。
- 「今行くべきスコア」の距離要素は座標が無いと計算できないため、MVP では距離はスコアから除外し、駅・エリア文字列ベースの弱い近接のみ扱う（[scoring](../specs/scoring.md)）。

## Alternatives Considered

- **Google Places 中心**: 精度は高いが課金が発生。無料方針に反するため将来オプション。
- **スクレイピング許容**: 精度は上がるが ToS 違反リスク・保守コスト大。却下。
- **手動入力のみ**: 最小実装だが「3 秒保存」の体感を損なう。OGP 自動補完を併用する。
