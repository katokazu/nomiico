# ADR 0003: 匿名ローカルユーザー → 任意昇格

## Status

Accepted (2026-06-29)

## Context

プロダクト思想は「入力は必要になるまで求めない」（[product-overview](../domain/product-overview.md) §3）。初回起動でログインを求めると保存体験が重くなる。一方、将来のグループ決定・端末間同期にはユーザー同一性が必要。

## Decision

**初回起動時に端末ローカルの匿名ユーザーを自動生成し、ログインなしで全機能を使えるようにする。アカウント連携（メール / OAuth）は後から任意で行える昇格パスとして設計する。**

- 初回起動で `owner_id`（UUID v4）を生成し、`app_user` テーブルと安全なストレージ（`expo-secure-store`）に保存する。
- すべてのユーザーデータはこの `owner_id` に紐づける（[ADR 0002](0002-local-first-with-sync-seam.md)）。
- リモート導入時は Supabase の匿名サインインを使い、同じ思想を維持したままメール / OAuth へリンクできるようにする。昇格時はローカル `owner_id` をリモート ID にマッピングして同期する。

## Consequences

- 初回保存までゼロ入力で到達できる。
- MVP ではログイン UI 自体が不要。設定画面に「アカウント連携（将来）」のプレースホルダのみ置く。
- 端末ローカル ID のため、昇格前は端末をまたいだ復元ができない。手動エクスポート/インポートで補完する。
- 昇格時のローカル↔リモート ID マッピングとマージ戦略は、リモート導入時に別途設計が要る。

## Alternatives Considered

- **アカウント必須**: 同期・グループは安定するが初回障壁が高く思想に反する。却下。
- **ログイン完全になし（owner_id も持たない）**: 実装は最小だが、将来の同期で所有境界を後付けするのが困難。`owner_id` は最初から持つ。
