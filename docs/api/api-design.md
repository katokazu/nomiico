# API設計案

## 状況

API実装はまだありません。

## 候補リソース

- 店舗（Restaurants）
- タグ（Tags）
- 訪問（Visits）
- 決定セッション（Decision sessions）
- 決定候補（Decision candidates）

## 設計原則

- 保存は部分的なデータでも許容する。
- 決定系エンドポイントは、すぐに役立つ候補を返す。
- APIでは、ユーザー自身の保存レコードと外部の店舗データを区別する。
